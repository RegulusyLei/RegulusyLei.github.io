webpack的使用(二)

一月 15, 2017 Study

加载器简介

webpack准备的一些预处理工具

加载器

1. babel-loader
    ++ 用来编译jsx和ES6到原生js
        -- jsx语法: 允许html和js混写
    ++ 安装
        npm install babel-loader --save-dev
        npm install babel-core babel-preset-es2015 babel-preset-react --save-dev
    ++ 修改配置
        -- 与入口和出口同级
        -- module: {
            loaders: [
                //用来处理jsx和es6语法
                {
                    test: /\.jsx?$/,
                    loader: 'babel',
                    query: {
                        presets: ['es2015', 'react']
                    }
                }
            ]
        }
        * query 是babel的插件, 上边已安装
    ++ 启动运行
2. css-loader
    ++ 运行在js中引用css文件
    ++ 安装
        npm install css-loader style-loader --save-dev
    ++ 修改配置
        -- 与上边一样,继续添加进loaders数组
        //处理css引用
        {
            test: /\.css$/,
            loader: 'style!css' //注意,多个加载器用感叹号连接,执行顺序是从右往左
        }
    ++ 启动运行
3. sass-loader
    ++ 用来编译sass文件
    ++ 安装
        npm install sass-loader --save-dev
    ++ 修改配置
        同上
        {
            test: /\.sass$/,
            loader: 'style!css!sass'
        }
    ++ 启动运行
4. url-loader
    ++ 用来处理图片的加载器
    ++ 安装
        npm install url-loader file-loader --save-dev
        * file-loader 已被url-loader内置
    ++ 修改配置
        同上
        {
            test: /\.(png|jpg|jpeg|gif)$/,
            loader: 'url?limit=25000'
        }
        * limit的作用: 如果图片大于给定的值, 图片就是指定的路径; 否则, 图片就转化成base64字符串.
        可以减少网络请求
    ++ 启动运行

webpack的使用(一)

一月 15, 2017 Study

webpack介绍

webpack类似gulp

使用webpack的框架

react
vue
angular 2

web快速开始

1. 创建项目目录
    dist目录
    src目录
    开发配置: webpack.develop.config.js
    发布配置: webpack.publish.config.js
2. 安装全局和本地
3. 修改配置文件
    var path = require('path');
    module.export = {
        //入口
        entry: path.resolve(_dirname,'src/js/app.js'),
        //出口
        output: {
            path: path.resolve(_dirname,'dist'),
            filename: 'bundle.js', //重命名
        },
    }
4. 启动
    webpack --config webpack.develop.config.js

快速改造

1. 命令太多太繁琐,每一步都要写命令.所以要快速改造一下配置文件,即:将命令配置到package.json文件中
    ++ 配置: 'develop': 'webpack --config webpack.develop.config.js'
    ++ 运行: npm run develop
2. 自动监控,自动构建
    ++ 工具: webpack-dev-server(第三方插件)
    ++ 配置: 
        webpack-dev-server --config webpack.develop.config.js --devtool eval --progress --colors --hot --content-base src
            1. webpack-dev-server: 封装了webpack命令
            2. --content-base src: 以src为根目录,启动一个服务器
3. 浏览器自动刷新
    ++ 配置: 
        //入口
        //新添加两行配置,还是依赖webpack-dev-server
        'webpack/hot/dev-server',
        'webpack-dev-server/client?http://localhost:8080',
        entry: path.resolve(_dirname,'src/js/app.js'),

Tips: dist文件夹删除后,依旧能运行,这是因为webpack帮忙构建的js在内存中

React介绍

一月 15, 2017 Study

简单来说, React是一个Javascript库.

++ 使用React制作的网站案例
    -- info.smartstudy.com

React四个概念

++ Virtual DOM
    -- 用来转换跨平台
    -- 可以让页面渲染速度提升(Diff算法)
++ React组件化
    -- 不同于从业务角度的模块化开发, 组件化开发是从UI角度出发
        --- 组件的四个特性
            1. 自定义标签: 可以定义Html中不存在的标签
            2. 高内聚的资源: 所有的资源写在同一个文件
            3. 独立的作用域: 使用ES6, Class具有独立作用域
            4. 规范化接口: 固定的规范接口, 不可更改
    -- 组件化的开发特性
        1. 可组合
        2. 可重用
        3. 可维护
        4. 可测试
++ Jsx语法
    -- 实质就是一个语法糖, 可以让组件结构看上去更清晰
    -- 允许html和js混写
    -- 解析规则: 碰到尖括号 '<' 就按html语法解析; 碰到大括号 '{' 就按javascript语法解析
++ Data Flow(单项数据流)
    -- 传统的MVC
    -- Flux
    -- Redux(Vue就是抄袭的这货, 较难)

React快速开始

++ 创建项目文件夹
    -- npm init //初始化npm配置文件
++ 依赖环境
    -- npm install --save webpack
